What's New

New Features

  • Skills pane — Browse, search, and install skills from the skills.sh marketplace directly inside Addy. View skill details, install counts, security audit status, and install or uninstall with one click.
  • Conversations pane — A unified view of all past chat and terminal conversations across Claude, Gemini, and Codex. Filter by source, scope, or category, search by keyword, and resume or fork any previous session.
  • Multi-model profiles with per-role assignments — Models are now full profiles with backend detection, capability flags, and model overrides. Assign different profiles for terminal, chat, helper (commit messages), and code review roles from the redesigned Models settings tab.
  • Git pane file context menu — Right-click any file in the Git pane to "Open in Finder" or "Delete File", in addition to the existing stage/unstage actions.
  • Sub-agent spawn direction setting — New preference (right, left, top, bottom) to control where sub-agent panes open relative to the parent.
  • Agent pane footer bar — Agent and chat panes now share a new footer component with the model picker and input field, replacing the old freestanding layout.

Improvements

  • Chat tool blocks collapse immediately — Completed tool blocks now roll up into a summary group after a short delay, instead of waiting for the next user message. Keeps the chat stream compact during long agent runs.
  • Terminal paste moved off main thread — Pasting into a terminal no longer blocks the UI. The pasteboard read happens on a background thread, preventing hangs when Universal Clipboard or clipboard managers are slow.
  • Commit message generation uses your helper profile — The "Generate commit message" feature now uses whichever model profile you've assigned as the helper, instead of hardcoding a path to the Claude binary.
  • Multiline task and plan prompts — The Tasks pane input and Plan prompt fields now support multi-line editing with auto-growing height, using the shared InlinePromptField component.
  • Tasks pane redesigned — Double-height header with a progress bar showing completion percentage, a summary label, and the new InlinePromptField for adding tasks.
  • Onboarding welcome screen refreshed — New centered hero layout with the Addy wordmark, cleaner typography, and simplified copy.
  • Settings window simplified — The "Interface" tab has been merged into "Appearance" (now includes pane defaults like default new-pane mode). Sidebar reduced to four tabs: Appearance, Models, Behavior, Integrations.
  • Settings window follows theme — The settings window background now updates live when you switch themes.
  • Model launch button rebuilt with ActionButton — The model picker button now uses the shared ActionButton component with proper dropdown menus, hover states, and active model display.
  • Pane split creation lag reduced — Splitting a pane in the same direction as its parent now inserts incrementally into the existing NSSplitView instead of tearing down and rebuilding the entire view hierarchy.
  • Terminal minimum size guard improved — The reflow-prevention guard now estimates actual column/row counts from cell dimensions instead of using a fixed pixel threshold, making it more accurate across different font sizes.
  • Runtime identity injected into all agent sessions — Every terminal, chat, and sub-agent pane now receives ADDY_RUNTIME, ADDY_SESSION_MODE, and ADDY_SOCKET environment variables so agents and MCP servers can reliably detect when they're running inside Addy.
  • Codex stream parser rewritten — Block start/complete events are now matched by item ID, fixing tool and text block mismatches that caused garbled chat output with Codex.
  • Gemini stream parser improved — Better multi-turn message ID tracking and content block correlation for more reliable chat rendering with Gemini CLI.
  • Code editor: Tab indents selection — Pressing Tab with a multi-line selection now indents all selected lines. Shift+Tab still dedents.
  • Code editor incremental highlighting — Syntax highlighting now only invalidates the changed range after edits, instead of re-highlighting the entire document.
  • Commit drawer auto-resizes — The git commit message drawer now adjusts its height as you type or when the field is cleared, preventing wasted space.
  • Pull button always enabled — The Git pane pull button is now always clickable (it was previously disabled when there were no remote changes), so you can manually fetch and pull at any time.
  • Plan build uses interactive CLI — The "Build" button on plans now opens an interactive CLI session instead of a non-interactive --print session, so you can guide the agent during execution.
  • App icon updated — New app icon design with refreshed gradients.

Bug Fixes

  • Notepad paste colors fixed in dark mode — Pasted text no longer retains invisible dark-on-dark foreground colors from the clipboard. A new paste sanitizer normalizes all text colors to the active theme palette.
  • Focus glow no longer clips on vertical splits — The focused pane glow effect was being clipped by parent split views when a pane lived inside a vertical (top/bottom) split. The glow container is now hosted on a layer that doesn't clip.
  • Code editor line numbers no longer reversed — Line numbers were rendering upside-down and scrolling in the wrong direction. Fixed by making the gutter view use a flipped coordinate system.
  • Skills pane no longer freezes on open — Opening the Skills pane could stall the main thread and freeze the launcher popover animation. Catalog loading is now fully asynchronous with a loading indicator.
  • Orphaned browser console panes fixed — Closing a browser pane now correctly closes its linked console pane, preventing orphaned consoles that couldn't be removed.
  • Commit message generation race condition fixed — Rapidly clicking "Generate" no longer causes stale responses to overwrite your commit message. Each request is tracked by a unique ID and discarded if superseded.
  • Expandable cards fill parent width — Expandable card views (used in reviews, plans, and skills) were sometimes drifting right instead of filling the available width. Fixed with correct constraint binding.
