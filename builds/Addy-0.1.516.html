What's New

New Features

  • Automatic browser console error surfacing — After editing browser-relevant files (.tsx, .ts, .jsx, .js, .css, .html, etc.), any console errors are now automatically shown inline to AI agents, so they can fix issues immediately without needing to manually check.
  • Home button in the browser pane — A new house icon button in the browser toolbar navigates back to the start page, useful after hitting a dead-end error page.

Improvements

  • Sub-agents no longer steal focus — Spawning a sub-agent (or review pane) no longer pulls focus away from your active pane. You can keep working while agents spin up in the background.
  • Click-to-focus now works reliably — Clicking any pane now properly makes it the focused/first-responder pane, not just the visually active one.
  • Git push/pull reliability — Push and pull operations now have a 30-second timeout, won't hang on credential prompts, and avoid pipe deadlocks that could freeze the UI.
  • Agent cursor always visible — CLI tools like Claude Code can no longer override the terminal cursor color, which previously made it invisible against the pane background.
  • CLI color output adapts to light/dark mode — Terminal panes now set the COLORFGBG environment variable so CLI tools automatically adjust their color output for your current theme.
  • Review agents work on more machines — Review agents and the architecture describer now launch through the user's login shell, fixing "command not found" failures on machines where Node.js is managed by nvm or Homebrew.
  • Git pane push button always visible — The Push button is now always shown (disabled when nothing to push), instead of disappearing and reappearing. Empty git sections have a cleaner styled placeholder.
  • Review panes open at better sizes — Code review panes start at 28% width. Plan review panes auto-pin at a compact 190px.
  • Sidebar toggle animation — Slightly faster (150ms) with an ease-out curve for a snappier feel.
  • Sub-agent connector lines — Now rendered at full opacity for better visibility.
  • Split button modifier changed — Ctrl+click (instead of Cmd+click) on split buttons now creates a split relative to the active pane.
  • Session telemetry — Now tracks session duration and pinned pane configuration for diagnostics.

Bug Fixes

  • Code editor: text now actually visible — Fixed a bug where the code editor pane showed a blank area with no text content due to missing background color and text attributes.
  • Code editor: line numbers work for all tabs — Line number gutter was rewritten as a standalone view, fixing an issue where only the first tab showed line numbers.
  • Code editor: tabs no longer scroll vertically — Tab bar vertical scroll elasticity disabled so tabs stay in a single horizontal row.
  • Code editor: clicking a tab no longer hides the header — Fixed a layout issue that caused the pane header to disappear when switching tabs.
  • Code editor: "+" creates an untitled file — The add-tab button now creates a new blank file instead of opening a confusing file chooser dialog.
  • Pin button spacing fixed — The pin button no longer overlaps the close button in pane headers (spacing corrected from +2 to -2).
  • Header controls reposition when pin button appears — Across all pane types (browser, console, file explorer, tests, design tokens, tasks, theme editor, architecture), custom header buttons now properly shift left when the pin button is installed, preventing visual overlap.
  • Light mode: segmented controller readable — Fixed overly dark segmented control backgrounds in light mode that made text unreadable.
  • File explorer opens at proper width — Fixed file explorer sometimes launching at ~40px width by pre-setting frame sizes before NSSplitView layout.
  • Auto-naming hook works inside agent sessions — The pane auto-name hook now clears CLAUDECODE env vars so it can run Claude CLI within an existing agent environment.
  • IPC timeouts aligned — Client timeout increased from 2s to 5s to match the server, reducing spurious timeout errors on slower operations.
